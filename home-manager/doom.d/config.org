** captureテンプレート
org-capture-templates のテンプレート本文（第5要素）は文字列 / (file ...) / (function ...) のいずれかしか受け付けない（org-capture-get-template参照）。
以前は本文を(concat ...) の生リストのまま登録していたため、いずれにもマッチせずorg-capture-get-templateが
フォールバックの "* Invalid capture template" を返してしまっていた。
バッククォート + アンクォートで concat を事前に評価し、プレーンな文字列にしてから登録することで解消する。

ファイル名生成時に read-string で聞いたタイトルを org-capture-put/get で capture の property list に
一時保存しておき、テンプレート本文側で %(org-capture-get :my-title) として読み出すことで #+TITLE: にも
流用する（同じタイトルを二度入力しなくて済む）。カーソル位置(%?)は本文の末尾に移し、そのままメモを書き始められるようにした。
#+begin_src emacs-lisp
(after! org
  (add-to-list 'org-capture-templates
        `("m" "Work memo" plain
          (file (lambda ()
                  (org-capture-put :my-title (read-string "Title: "))
                  (expand-file-name
                   (concat (format-time-string "%Y%m%d-")
                           (org-capture-get :my-title)
                           ".org")
                   org-directory)))
          ,(concat ":PROPERTIES:\n"
                  ":ID: %(org-id-new)\n"
                  ":END:\n"
                  "#+TITLE: %(org-capture-get :my-title)\n"
                  "#+OPTIONS: broken-links:t toc:nil \\n:t ^:nil num:nil\n"
                  "#+DATE: %<%Y-%m-%d>\n"
                  "#+LANGUAGE: ja\n"
                  "\n%?")
          :jump-to-captured t)))
#+end_src

** org-roamノード作成時に保存先ディレクトリを選択する
org-roam-directory を org-directory 全体に広げたことで、ノードをどのサブディレクトリに
置くか毎回手動で mkdir/mv する必要が出てきた。org-roam-node-find から新規ノードを
作成する際に、既存のサブディレクトリを補完候補として選ぶ（TABで既存の中から選択、
自由入力で新規ディレクトリを作成、空欄でルート直下）ようにして手間を省く。
#+begin_src emacs-lisp
(defun my/org-roam--select-directory ()
  "org-roam-directory 配下の既存サブディレクトリを選択、または新規ディレクトリ名を入力する。
空文字のまま確定するとルート直下に作成する。"
  (let* ((subdirs (thread-last
                     (directory-files-recursively org-roam-directory "" t)
                     (seq-filter #'file-directory-p)
                     (mapcar (lambda (d) (file-relative-name d org-roam-directory)))
                     (cons "")
                     (delete-dups)))
         (choice (completing-read "Directory (空欄でルート直下): " subdirs nil nil)))
    (if (string-empty-p choice)
        org-roam-directory
      (let ((dir (expand-file-name choice org-roam-directory)))
        (make-directory dir t)
        dir))))

(after! org-roam
  (setq org-roam-capture-templates
        '(("d" "default" plain "%?"
           :target (file+head "%(my/org-roam--select-directory)/%<%Y%m%d>-${slug}.org"
                               ":PROPERTIES:\n:ID:       %(org-id-new)\n:END:\n#+TITLE: ${title}\n#+OPTIONS: broken-links:t toc:nil \\n:t ^:nil num:nil\n#+DATE: %<%Y-%m-%d>\n#+LANGUAGE: ja\n")
           :unnarrowed t))))
#+end_src

** 保存時に自動export（AUTO_EXPORTキーワード）
個別のorgファイル側で ~#+AUTO_EXPORT: md~ のように書いておくと、保存時に
自動でそのバックエンドへexportする汎用フック。スペース区切りで複数指定可能
（例: ~#+AUTO_EXPORT: md html~）。キーワードを書かないファイルには一切影響しない。

~"md"~ はorg組み込みの ~org-md-export-to-markdown~ ではなく、pandoc経由の
GFM（GitHub Flavored Markdown）exportにしている。組み込みの方はテーブルを
markdown記法に変換できず生のHTMLタグをそのまま埋め込んでしまうため
（ox-mdの既知の制限）。pandocのgfm exportは拡張子も ~.md~ のまま
（~org-pandoc-output-format-extensions~ で ~gfm~ は ~md~ に対応）で、
テーブルもきちんとpipe記法に変換される。
#+begin_src emacs-lisp
(defvar my/org-auto-export-backends
  '(("md"      . org-pandoc-export-to-gfm)
    ("html"    . org-html-export-to-html)
    ("gfm"     . org-pandoc-export-to-gfm)
    ("docx"    . org-pandoc-export-to-docx))
  "AUTO_EXPORTキーワードの値と対応するexport関数の対応表。
新しいバックエンドを増やしたい時はここに追加する。")

(defun my/org-auto-export-if-requested ()
  "バッファ先頭に #+AUTO_EXPORT: があれば、保存時に自動でexportする。"
  (when (derived-mode-p 'org-mode)
    (when-let* ((keyword (cadr (assoc "AUTO_EXPORT"
                                       (org-collect-keywords '("AUTO_EXPORT")))))
                (backends (split-string keyword)))
      (dolist (name backends)
        (if-let ((fn (cdr (assoc name my/org-auto-export-backends))))
            (progn
              (funcall fn)
              (message "AUTO_EXPORT: %s へexportしました" name))
          (message "AUTO_EXPORT: 未知のバックエンド %s" name))))))

(add-hook 'after-save-hook #'my/org-auto-export-if-requested)
#+end_src

** HTMLホットリロードプレビュー（impatient-mode）
現在のorgバッファをexportしたHTMLを、実ブラウザ（eww等ではなくChrome/Safari）で
ホットリロードしながらプレビューする。~simple-httpd~ でローカルHTTPサーバーを
立て、~impatient-mode~ がバッファの変更を検知してブラウザ側へpush通知する仕組み。
#+begin_src emacs-lisp
(use-package! impatient-mode
  :commands (impatient-mode imp-visit-buffer imp-set-user-filter))

(after! impatient-mode
  ;; 編集のたびに毎回exportすると重いので、0.5秒の無操作を挟んでから更新する
  (setq impatient-mode-delay 0.5))

(defun my/imp-org-html-filter (buffer)
  "orgバッファの内容をHTMLへexportしてimpatient-modeに渡すフィルタ。
imp--send-stateからstandard-outputが出力先バッファにbindされた状態で呼ばれるため、
princでそこに書き込む。"
  (princ (with-current-buffer buffer
           (org-export-as 'html nil nil nil))))

(defun my/org-impatient-preview ()
  "現在のorgバッファを、実ブラウザでホットリロードしながらプレビューする。"
  (interactive)
  (unless (derived-mode-p 'org-mode)
    (user-error "org-modeバッファで実行してください"))
  (imp-set-user-filter #'my/imp-org-html-filter)
  (imp-visit-buffer))

(map! :map org-mode-map
      :localleader
      :desc "Live HTML preview" "P" #'my/org-impatient-preview)
#+end_src

** file:リンクのエクスポート挙動
file:リンクはエクスポート時にリンクではなく説明文（無ければパス）だけを出力する。
ローカルの絶対パスはエクスポート先（Markdown/HTMLなど）では意味を持たないため。
file以外のリンクタイプ（http, https, idなど）には影響しない。
#+begin_src emacs-lisp
(after! org
  (org-link-set-parameters "file"
    :export (lambda (link description _format _info)
              (or description link))))
#+end_src

file:リンクに検索オプション（::33や::send_messageなど）が付いていると、
org-html-linkなどが上のexport上書きより先に検索文字列の解決を試みてしまい、
見つからない場合はorg-export-with-broken-linksによりエクスポートが中断される。
壊れた内部リンク（見出し/IDが解決できないリンク）があった場合はエクスポートを中断する（デフォルト値を明示）。
#+begin_src emacs-lisp
(setq org-export-with-broken-links nil)
#+end_src

保留中の対応案（エクスポート直前にパースツリーからfile:リンクの:search-optionだけ取り除いて回避する）。
今は使っていないので tangle しない。
#+begin_src emacs-lisp :tangle no
(defun my/org-strip-file-link-search-options (tree _backend _info)
  (org-element-map tree 'link
    (lambda (link)
      (when (equal (org-element-property :type link) "file")
        (org-element-put-property link :search-option nil))))
  tree)
(add-to-list 'org-export-filter-parse-tree-functions
             #'my/org-strip-file-link-search-options)
#+end_src

** org-modernを無効化する（+pretty）
